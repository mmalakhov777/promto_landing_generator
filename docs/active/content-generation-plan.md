## Шаг 1 — Подключить backend к существующей БД

Создать миграции Alembic и обновить `DATABASE_URL`.

### 1.1 Создать Alembic миграции

```bash
cd backend
# Инициализировать alembic если не инициализирован
alembic init alembic
```

Сгенерировать миграцию для всех таблиц:
- `categories`
- `landings`
- `landing_contents`
- `landing_sections`
- `site_settings`
- `users`
- `slug_redirects`

### 1.2 Настроить DATABASE_URL

В `.env` backend:
```
DATABASE_URL=postgresql+asyncpg://gen_user:n%40%21fOXG%7CP%2AlQ9@176.53.163.226:5432/default_db
```

### 1.3 Проверить что миграции не трогают существующие таблицы блога

Таблицы проекта — с префиксом, который НЕ используется блогом.
Использовать схему `public` (по умолчанию) — убедиться что имена таблиц (`landings`, `categories` и т.д.) не конфликтуют с таблицами блога.

**Проверка:** подключиться к БД и посмотреть список существующих таблиц:
```sql
SELECT table_name FROM information_schema.tables WHERE table_schema = 'public';
```

## Шаг 3 — Генерировать контент (в рамках чата)

Для КАЖДОГО лендинга — сгенерировать вручную:

### Для каждого лендинга создать:

**SEO:**
- `meta_title` (< 60 симв., с ключевым словом)
- `meta_description` (< 160 симв., с ключевым словом)
- `h1` (с ключевым словом, до 60 симв.)
- `og_title`, `og_description`

**Hero:**
- `hero_title` (= h1)
- `hero_subtitle` (1-2 предложения о выгоде)
- `hero_cta_text` ("Создать сайт")
- `hero_placeholder` ("Опишите, какой сайт для {ниша}...")

**Преимущества:**
- 4-6 карточек с иконками (zap, shield, star, clock, globe, lock...)
- Адаптированы под нишу

**Как это работает:**
- 3-4 шага (описал → ИИ создал → опубликовал)

**FAQ:**
- 5-6 вопросов (3 нишевых + 2 общих о Промто)

**Отзывы:**
- 4-5 шаблонных отзывов

**CTA тексты:**
- `cta_mid_title`, `cta_mid_subtitle`
- `cta_final_title`, `cta_final_subtitle`, `cta_final_button_text`

## Формат хранения контента для方便ной работы

Контент удобно хранить в Structured Query для быстрого COPY в базу:

```sql
UPDATE landing_contents
SET
  meta_title = '...',
  meta_description = '...',
  ...
WHERE landing_id = X AND locale = 'ru';
```
